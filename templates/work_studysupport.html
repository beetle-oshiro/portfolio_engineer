Python / Flask / ChatGPT API / PostgreSQL

アシスト付きワード保存

ChatGPTAPIとFlaskとPostgreSQLを用いて、ワードや詳細を検索・登録（ChatGPTAPIによるアシスト有）・更新・削除するシステムを制作しました。

※無料版のFlaskを使用しているので、アクセスしてすぐはスリープ状態になっています。少し経ってから見れるようになります。数分触らないでいてもスリープ状態になります。

作品を開く

制作目的

  • ChatGPTにほとんどの作業を任せた場合、どれくらい簡単に、どのようなものを作成できるのか興味がありました。
  • 学習中に出会った「プログラミング用語」や「コードの説明」を効率よく整理・記録するために、 自分専用の辞書アプリを作りたいと考え、ChatGPTAPIと連携した学習支援ツールを開発しました。
  • このアプリでは、自分で入力するだけでなく、ChatGPTに説明やコードを自動生成させることも可能で、 「記録するだけで終わらない、学び直しに強い」学習アシスタントを目指しました。

制作の流れ

  1. この制作物はChatGPTに全て制作してもらおうと考えていたので、始めに「制作する目的」や「何ができるものか」、「どういうページ構成か」を伝えて、1ページずつ制作していきました。
  2. 基本的にはChatGPTが出してくれるコードをVSCodeにコピペして動作確認し、エラーが出た時もChatGPTに相談しながら解決していきました。

操作方法

  1. 「ログイン名（tanobi_test_login）」と「パスワード（testpass456）」を入力してログイン。
  2. ログイン後、「登録 or 検索」のモードを選択
  3. 「登録」を選ぶと、登録画面で以下を入力・選択：
    • 学習ワード、詳細説明、タグなど
    • 「説明をアシスト」や「コードをアシスト」にチェックを入れると、ChatGPTがワードに対する簡単な説明とコードを自動生成
  4. 「検索」を選ぶと、検索画面で「検索ワード」、「検索方法」、「検索対象」、「タグ」を入力・選択して検索をします。
  5. 検索結果からデータの編集・削除ができます。
  6. 管理者（ログイン名：admin、パスワード：admintest）は、ユーザーやタグの登録・編集・削除ができます。

感じたことや反省点

  • コードを全てChatGPTに任せたので、私が作成する何倍も早くカタチになりました。
  • コーディングに入る前にWhimsicalというアプリを用いて仮の設計のようなものを作りましたが、実際に制作を始めると設計通りに進まない部分が多々あり、設計の段階でもっと時間をかけるべきだったと反省しています。
  • 同じようなことをさせる処理でも翌日には違う手法のコードを出してくるので、全体で見ると統一感の無いコードになってしまいました。
  • より早く、より精度の良いものを作るためには、ChatGPTに様々な指定をする必要があると感じました。
  • ChatGPTAPIはデータベースやネットに直接アクセスすることができないと知りました。
