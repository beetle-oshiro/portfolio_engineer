Python / Flask / ChatGPT API / PostgreSQL

ワード保存

まだ製作途中ですが、ワードや詳細といった項目のデータをデータベースに登録・編集・検索できるようにしています。

※無料版のFlaskを使用しているので、アクセスしてすぐはスリープ状態になっています。少し経ってから見れるようになります。数分触らないでいてもスリープ状態になります。

作品を開く

制作目的

  • 学習しながらワードの意味等を保存しておける自分用のサイト（システム）を制作しようと思いました。
  • ChatGPTを利用すると、初めて挑戦する言語でも何か作れるのか興味があり、挑戦してみました。

制作の流れ

  1. 最終的にはChatGPT APIと連携してみたいと思っているので、その点も踏まえてChatGPTに相談したところ言語はPythonにしようという流れになりました。
  2. 初めはPythonの本を読んで基礎から勉強を始めましたが、わからない部分をChatGPTに質問しているうちに「もう全部ChatGPTから教わった方が早いのではないか？」と思うようになり、イチからChatGPTに教わる方向にしました。

操作方法

登録

  1. トップページで「登録」ボタンを押します。
  2. 「ワード」、「詳細」、「タグ」、「状況」、「メモ」を入力（選択）し、「記録する」ボタンを押します。

検索

  1. トップページで「検索」ボタンを押します。
  2. 登録されているデータが一覧表示されるので、「タグ」か「キーワード」を入力（選択）し、「検索」を押します。

タグの管理

  1. トップページで「タグの管理へ」を押します。
  2. タグ名を入力して「追加」ボタンを押すことで、登録時に選択するタグを追加します。

今後の改善点

レスポンシブ対応とデザインを整える。

感じたことや反省点

  • ChatGPTだとわからない部分をすぐに質問できるので、本を読みながら制作したり、わからない部分をネットで調べていた頃よりはスムーズに学習が進みました。
  • 数日学習した辺りから、私自身の脳に記憶されている感覚が薄いことに気づきました。
  • いつの間にか私がやりたいことを伝えてChatGPTがコードを書く流れになり、自分で制作したといっていいのか悩ましい制作物になってしまいました。
  • GitHub・Render・pgAdminもChatGPTに教わりながら登録しましたが、こちらに関してはChatGPTのおかげでスムーズにできたと感じています。
  • ChatGPTの書くコードでエラーは出ませんが、少し時間を置いて同じ部分をお願いすると前回と違ったコードを書いたり、ChatGPTの指示通りにした部分を数日後のChatGPTが否定したりするので、協力の仕方も工夫がいるなと感じました。
  • 現状の私のスキルではChatGPTを上手く使いこなす方が早く制作物が仕上がりますが、自分の身になっているのか凄く怪しいので、プロの方にChatGPTを使うバランスを質問してみたいです。
